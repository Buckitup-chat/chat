File Sandbox

Client-side file upload/download with AES-256-GCM encryption + ML-DSA-87 signing

← Electric Index

Documentation

Upload Protocol

  1. Import ML-DSA-87 keys (from user sandbox export)
  2. Select file → chunk into 4 MB pieces
  3. For each chunk: encrypt (AES-256-GCM), sign (ML-DSA-87)
  4. GET challenge, sign it, POST chunk mutation with auth to ingest
  5. Build manifest with chunk_sign_hashes, sign, POST to ingest
  6. Save the encryption secret for later download

Download Protocol

  1. Enter file_id and encryption secret (hex)
  2. Fetch file manifest via Electric Shape API
  3. Fetch all chunks, validate count matches manifest
  4. Decrypt each chunk with AES-256-GCM using enc_secret
  5. Reassemble and trigger browser download

No signing keys required — only the encryption secret.

file_chunks Schema

file_id        TEXT        "f_" + UUID
chunk_index    INTEGER     0-based position
data_b64       BYTEA       nonce(12)||ciphertext||tag(16)
size           INTEGER     encrypted chunk byte size
uploader_hash  TEXT        "u_" + hex(SHA3-512(sign_pkey))
owner_timestamp BIGINT     monotonic counter
sign_b64       BYTEA       ML-DSA-87 signature

files Schema (Manifest)

file_id           TEXT      "f_" + UUID
uploader_hash     TEXT      FK to user_cards
total_size        BIGINT    plaintext file size
chunk_size        INTEGER   4194304 (4 MB)
chunk_count       INTEGER   total chunks
chunk_sign_hashes BYTEA[]   SHA3-512(chunk.sign_b64)
owner_timestamp   BIGINT    monotonic counter
deleted_flag      BOOLEAN   soft delete
sign_b64          BYTEA     ML-DSA-87 signature

Encryption

AES-256-GCM with random 12-byte nonce. Each chunk stored as nonce(12) || ciphertext || tag(16). A unique 32-byte enc_secret is generated per file.

Signing

ML-DSA-87 (FIPS 204). Signature payload: all fields except sign_b64, sorted alphabetically, encoded per suffix rules, concatenated.

  • _b64, _cert, _pkey → padded Base64
  • Arrays → elements concatenated (Base64 if binary)
  • Booleans → "true" / "false"
  • Numbers → string representation
  • Strings → as-is

Challenge-Response Auth

  1. GET /electric/v1/challenge → {challenge, challenge_id}
  2. Sign challenge string with ML-DSA-87
  3. POST to /electric/v1/ingest with mutations + auth payload
